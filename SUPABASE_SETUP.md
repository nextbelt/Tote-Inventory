# Supabase Setup Guide

## 1. Create a Supabase Project

1. Go to [supabase.com](https://supabase.com) and sign up
2. Create a new project called **tote-inventory**
3. Copy your **Project URL** and **Anon Public Key** from **Settings → API**

## 2. Configure Environment Variables

Create a `.env` file in the project root (it's already git-ignored):

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key-here
```

> ⚠️ **Never** commit `.env` to version control.

## 3. Create the Database Table

In Supabase dashboard → **SQL Editor**, run the contents of [`database-setup.sql`](database-setup.sql):

```sql
CREATE TABLE totes (
  id         BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  name       TEXT NOT NULL CHECK (char_length(name) <= 100),
  position   TEXT NOT NULL UNIQUE,
  items      JSONB DEFAULT '[]'::jsonb,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 4. Create Storage Bucket

1. Go to **Storage → New bucket**
2. Name: `tote-images`, set to **Public**
3. Add policies for SELECT, INSERT, and DELETE — set each to `true` for development

> ⚠️ For production, restrict storage policies to authenticated users.

## 5. Enable Real-time

Run in SQL Editor:

```sql
ALTER PUBLICATION supabase_realtime ADD TABLE totes;
```

## Free Tier Limits

| Resource  | Limit                  |
|-----------|------------------------|
| Database  | 500 MB                 |
| Storage   | 1 GB (~300-500 photos) |
| Bandwidth | 5 GB/month             |
| Requests  | 2 M/month              |

## Deployment

When deploying to Netlify, Vercel, or similar, add these **environment variables** in the platform dashboard:

- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_ANON_KEY`

The app gracefully falls back to localStorage when Supabase is unavailable.
